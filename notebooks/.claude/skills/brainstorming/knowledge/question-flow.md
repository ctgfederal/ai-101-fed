# Question Flow

## Clarity Assessment

Read the user's opening statement. Classify:

- **Clear** — vision, audience, and outcomes already stated. Recommend skipping to `/build` or `/specify`. Capture only the gaps.
- **Directional** — goal known, but scope/audience/outcomes unclear. Light brainstorm: ask 3–5 of the 8 questions.
- **Exploratory** — vague spark, no concrete shape. Full 8-question walk.

If unsure, ask one calibration question: "Is the goal here clear, or are we still shaping the why?"

## Question Progression (8 ordered)

See `templates/question-progression.md`. Adapt to user energy:

- **Skip** what they've already answered.
- **Dig deeper** where they're animated — follow that thread.
- **Don't batch.** One question at a time. Wait. Listen.

## Stop conditions
- Vision is clear (you can summarize it back accurately).
- User says "proceed", "I'm done", or asks for next step.
- User repeatedly answers "I don't know" — capture as an open question and stop.

## Steering away from HOW

If the user proposes technical or design choices ("we should use Postgres", "let's build it in Rails"), redirect:

> "Good instinct — capture that as an open question. We'll work it in `/build` where we walk every design decision with tradeoffs."

This is `/brainstorm`, not `/build`.
